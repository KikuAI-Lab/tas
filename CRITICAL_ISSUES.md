# Критические проблемы детекции спама

## Обнаруженные проблемы

### False Positives (21.7% - КРИТИЧНО!)
1. "Мы покупаем продукты" → блокируется (score: 0.55)
2. "Ищу работу программистом" → блокируется (score: 0.65)
3. "Звоните мне завтра" → блокируется (score: 0.65)
4. "Продали дом в прошлом году" → блокируется (score: 0.55)
5. "Я продаю дом в прошлом году" → блокируется (score: 0.60)

### Причины

1. **Слишком широкие паттерны:**
   - "Commercial trade offer" ловит "покупаем", "ищу", "продали" в любом контексте
   - "Job offer" ловит "работа" даже когда это не предложение работы
   - "звоните" ловит любые упоминания звонков

2. **Низкий threshold (0.55):**
   - Паттерны с score 0.40 + бусты легко превышают threshold
   - Не учитывается контекст (прошедшее время, "я/мы" vs предложение)

3. **Агрессивные бусты:**
   - Буст для коротких коммерческих сообщений (+0.15)
   - Буст для "работа" в коротких сообщениях (+0.10)

## Решение

1. Повысить threshold до 0.65-0.70
2. Улучшить паттерны - добавить отрицательный контекст
3. Убрать/ослабить агрессивные бусты
4. Добавить whitelist паттернов (отрицательный контекст)
